## What Are Blog Posts?

Blog posts are articles that appear in the news/blog section of your website. They're perfect for sharing updates, health tips, practice news, and community information with your patients and visitors.

## Creating a Blog Post

### Step-by-Step Guide

<Steps>
  <Step title="Access Blog Posts">
    1. Log into the admin panel
    2. Click **Content** in the left sidebar
    3. Select **Blog Posts**
    4. Click the **Create Item** button
    
    <img src="/images/content/create-blog-post-button.png" alt="Create Item button in the blog posts section" />
  </Step>

  <Step title="Add Basic Information">
    Fill in the essential details:
    
    **Title** (Required)
    - Keep it clear and descriptive
    - Under 60 characters for SEO
    - Include relevant keywords naturally
    
    **Slug** (Required)
    - Auto-generated from title
    - Edit if needed for clarity
    - Use hyphens between words
    
    <img src="/images/content/blog-title-slug.png" alt="Title and slug fields in the blog post form" />
  </Step>

  <Step title="Write Your Content">
    Use the rich text editor to create your content:
    
    <img src="/images/content/rich-text-editor.png" alt="Rich text editor showing formatting toolbar and content area" />
    
    **Formatting Tools Available:**
    - **Bold** and *italic* text
    - Headings (H2, H3, H4)
    - Bullet points and numbered lists
    - Links to other pages or external sites
    - Block quotes for emphasis
    - Code blocks for technical content
  </Step>

  <Step title="Add a Featured Image">
    1. Click the **Featured Image** field
    2. Choose **Upload Files** or select from existing images
    3. Add descriptive **Alt Text** (required for accessibility)
    4. Optionally add a caption
    
    <img src="/images/content/featured-image-upload.png" alt="Featured image upload dialog with alt text field" />
    
    <Tip>
      Featured images appear at the top of your blog post and in blog listings. Choose high-quality, relevant images.
    </Tip>
  </Step>

  <Step title="Set Publication Details">
    Configure when and how your post appears:
    
    **Author** - Your name or the content creator
    **Published Date** - When the post should go live
    **Category** - Group related posts together
    **Status** - Draft, In Review, or Published
    
    <img src="/images/content/publication-details.png" alt="Publication details form with author, date, and status fields" />
  </Step>

  <Step title="Add SEO Information">
    Help search engines understand your content:
    
    **Excerpt** - Brief summary (150-160 characters)
    **Meta Title** - SEO title (if different from main title)
    **Meta Description** - Search result description
    
    <img src="/images/content/seo-fields.png" alt="SEO fields including excerpt and meta description" />
  </Step>

  <Step title="Save and Publish">
    Choose your next action:
    - **Save as Draft** - Keep working on it later
    - **Submit for Review** - Send to reviewers
    - **Publish** - Make it live immediately (if you have permission)
    
    <img src="/images/content/save-publish-options.png" alt="Save and publish buttons with status dropdown" />
  </Step>
</Steps>

## Rich Text Editor Guide

### Formatting Your Content

<AccordionGroup>
  <Accordion title="üìù Text Formatting">
    **Basic Formatting:**
    - **Bold**: Use for emphasis and key points
    - *Italic*: Use for subtle emphasis or foreign words
    - ~~Strikethrough~~: Use sparingly for corrections
    
    **Best Practices:**
    - Don't overuse formatting
    - Be consistent throughout your content
    - Use formatting to improve readability
    
    <img src="/images/content/text-formatting.png" alt="Text formatting options in the editor toolbar" />
  </Accordion>

  <Accordion title="üìã Lists and Structure">
    **Bullet Points:**
    - Use for unordered information
    - Keep items parallel in structure
    - Limit to 7-9 items per list
    
    **Numbered Lists:**
    1. Use for step-by-step instructions
    2. Use for ranked or ordered information
    3. Use for processes and procedures
    
    **Headings:**
    - Use H2 for main sections
    - Use H3 for subsections
    - Keep hierarchy logical
    
    <img src="/images/content/lists-headings.png" alt="List and heading options in the editor" />
  </Accordion>

  <Accordion title="üîó Links and References">
    **Internal Links:**
    - Link to other pages on your website
    - Use descriptive link text ("Read our services page" not "Click here")
    - Check links work before publishing
    
    **External Links:**
    - Link to reputable sources
    - Set to open in new tab for external sites
    - Verify links are current and working
    
    **Creating Links:**
    1. Select the text you want to link
    2. Click the link button or press Ctrl+K
    3. Enter the URL or search for internal pages
    4. Choose whether to open in new tab
    
    <img src="/images/content/link-creation.png" alt="Link creation dialog with URL field and options" />
  </Accordion>

  <Accordion title="üñºÔ∏è Images and Media">
    **Inserting Images:**
    1. Place cursor where you want the image
    2. Click the image button in the toolbar
    3. Upload new image or select existing
    4. Add alt text and caption
    5. Choose alignment (left, center, right)
    
    **Image Best Practices:**
    - Use high-quality, relevant images
    - Always add descriptive alt text
    - Keep file sizes reasonable (under 2MB)
    - Use consistent image sizes
    
    <img src="/images/content/image-insertion.png" alt="Image insertion dialog with upload and alignment options" />
  </Accordion>
</AccordionGroup>

## Blog Post Fields Explained

### Required Fields

<Tabs>
  <Tab title="Title">
    **Purpose**: The main headline of your blog post
    
    **Guidelines**:
    - Keep under 60 characters for SEO
    - Make it descriptive and engaging
    - Include relevant keywords naturally
    - Avoid clickbait or misleading titles
    
    **Examples**:
    ‚úÖ "New Diabetes Support Group Starting in March"
    ‚úÖ "5 Tips for Managing Seasonal Allergies"
    ‚ùå "You Won't Believe What Happened Next"
    ‚ùå "URGENT: READ THIS NOW!!!"
  </Tab>

  <Tab title="Slug">
    **Purpose**: The URL-friendly version of your title
    
    **Guidelines**:
    - Auto-generated from title but can be edited
    - Use lowercase letters and hyphens
    - Keep it short but descriptive
    - Avoid special characters
    
    **Examples**:
    ‚úÖ "diabetes-support-group-march"
    ‚úÖ "seasonal-allergy-tips"
    ‚ùå "New Diabetes Support Group Starting in March 2024!!!"
    ‚ùå "tips_for_allergies"
  </Tab>

  <Tab title="Content">
    **Purpose**: The main body of your blog post
    
    **Guidelines**:
    - Start with the most important information
    - Use headings to break up long sections
    - Include bullet points and lists
    - Write in a conversational, helpful tone
    - Aim for 300-800 words for most posts
    
    **Structure Template**:
    1. Opening paragraph (what this post is about)
    2. Main content (detailed information)
    3. Conclusion or call-to-action
  </Tab>

  <Tab title="Status">
    **Purpose**: Controls whether the post is visible to the public
    
    **Options**:
    - **Draft**: Work in progress, only you can see it
    - **In Review**: Submitted for approval, reviewers can see it
    - **Published**: Live on the website, everyone can see it
    
    **Workflow**:
    Draft ‚Üí In Review ‚Üí Published
  </Tab>
</Tabs>

## Content Ideas and Templates

### Health Tips Template

```markdown
# [Condition/Topic] Tips for [Season/Situation]

## Introduction
Brief explanation of why this topic is important now.

## Key Tips
1. **Tip 1**: Detailed explanation
2. **Tip 2**: Detailed explanation
3. **Tip 3**: Detailed explanation

## When to Seek Help
- Symptom 1
- Symptom 2
- Symptom 3

## Contact Information
How to reach the practice for more information.
```

### Practice Update Template

```markdown
# [Update Type]: [Brief Description]

## What's Changing
Clear explanation of the change.

## When It Takes Effect
Specific dates and timeline.

## How It Affects You
Impact on patients and services.

## What You Need to Do
Any actions required from patients.

## Questions?
Contact information for more details.
```

### Event Announcement Template

```markdown
# [Event Name] - [Date]

## Event Overview
What the event is and why it's important.

## Event Details
- **Date**: [Date]
- **Time**: [Time]
- **Location**: [Location]
- **Cost**: [Free/Cost]

## Who Should Attend
Target audience description.

## How to Register
Registration process and deadline.

## More Information
Contact details for questions.
```

  <Step title="Use Search and Filters">
    - **Search**: Type keywords to find specific posts
    - **Filter by Status**: Show only drafts, published, etc.

    - **Filter by Date**: Find posts from specific time periods
    
    <img src="/images/content/blog-search-filters.png" alt="Search bar and filter options in blog post list" />

  <Step title="Edit the Post">
    Click on the post title or edit icon to open the editor

  <Step title="Make Your Changes">
    Edit any fields as needed
  </Step>
  
  <Step title="Save Changes">
    Click "Save" to preserve your edits
  </Step>

### Bulk Operations

<img src="/images/content/bulk-operations.png" alt="Bulk selection checkboxes and action buttons" />

**Available Bulk Actions:**
- **Delete Multiple Posts**: Remove several posts at once
- **Change Status**: Move multiple posts to draft/published
- **Export**: Download post data
- **Duplicate**: Create copies of existing posts

**How to Use Bulk Operations:**
1. Check the boxes next to posts you want to modify
2. Click the bulk action button that appears
3. Choose your action from the dropdown
4. Confirm the operation

### Version History

<img src="/images/content/version-history.png" alt="Version history panel showing previous edits and timestamps" />

**Track Changes:**
- See who made changes and when
- Compare different versions
- Restore previous versions if needed

**Accessing Version History:**
1. Open any blog post for editing
2. Look for "Revisions" or "History" tab
3. Click to see all previous versions
4. Select a version to compare or restore

## Common Issues and Solutions

### Troubleshooting

<AccordionGroup>
  <Accordion title="üö´ Can't Publish Content">
    **Possible Causes:**
    - You don't have publishing permissions
    - Required fields are missing
    - Content is still in draft status
    
    **Solutions:**
    - Check with your administrator about permissions
    - Ensure all required fields are filled
    - Change status to "Published" before saving
    - Submit for review if you can't publish directly
  </Accordion>

  <Accordion title="üñºÔ∏è Images Not Displaying">
    **Possible Causes:**
    - Image file is too large
    - Unsupported file format
    - Missing alt text
    - Upload failed
    
    **Solutions:**
    - Resize images to under 2MB
    - Use JPG, PNG, or WebP formats
    - Always add alt text
    - Try uploading again
    - Check your internet connection
  </Accordion>

  <Accordion title="üîó Links Not Working">
    **Possible Causes:**
    - Incorrect URL format
    - Link to unpublished content
    - External site is down
    - Typo in the URL
    
    **Solutions:**
    - Double-check URL spelling
    - Ensure linked content is published
    - Test external links in a new tab
    - Use the link picker for internal pages
  </Accordion>

  <Accordion title="üíæ Content Not Saving">
    **Possible Causes:**
    - Internet connection issues
    - Session timeout
    - Browser problems
    - Server issues
    
    **Solutions:**
    - Check your internet connection
    - Log out and log back in
    - Try a different browser
    - Copy your content before refreshing
    - Contact support if problems persist
  </Accordion>
</AccordionGroup>
