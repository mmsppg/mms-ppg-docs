## What Are Events?

Events are scheduled activities, meetings, workshops, or gatherings that you want to promote to your community. They appear in a dedicated events section and can include registration information, location details, and calendar integration.

## Creating an Event

### Step-by-Step Guide

<Steps>
  <Step title="Access Events Section">
    1. Log into the admin panel
    2. Click **Content** in the left sidebar
    3. Select **Events**
    4. Click the **Create Item** button
    
    <img src="/images/content/create-event-button.png" alt="Create Item button in the events section" />
  </Step>

  <Step title="Add Event Basics">
    Fill in the essential information:
    
    **Title** (Required)
    - Clear, descriptive event name
    - Include key details if helpful
    - Keep it concise but informative
    
    **Slug** (Required)
    - Auto-generated from title
    - Edit for clarity if needed
    - Use hyphens between words
    
    <img src="/images/content/event-title-slug.png" alt="Event title and slug fields" />
    
    **Examples:**
    ‚úÖ "Monthly PPG Meeting - March 2024"
    ‚úÖ "Diabetes Support Workshop"
    ‚úÖ "Community Health Fair"
  </Step>

  <Step title="Set Date and Time">
    Configure when your event happens:
    
    **Event Date** (Required)
    - Use the date picker
    - Double-check the date is correct
    - Consider day of the week
    
    **Event Time** (Optional)
    - Use 24-hour or 12-hour format
    - Include start time at minimum
    - Add end time if helpful
    
    <img src="/images/content/event-datetime.png" alt="Date and time picker for events" />
    
    <Tip>
      Consider your audience when choosing dates and times. Avoid conflicts with holidays or other major community events.
    </Tip>
  </Step>

  <Step title="Add Location Information">
    Help people find your event:
    
    **Location** (Optional but recommended)
    - Full address including postcode
    - Building or room name if applicable
    - Parking information if relevant
    
    **Examples:**
    - "MMS Surgery, 123 High Street, Anytown, AB1 2CD"
    - "Community Centre, Main Hall, 456 Church Lane"
    - "Online via Zoom (link will be sent to registered participants)"
    
    <img src="/images/content/event-location.png" alt="Location field with address information" />
  </Step>

  <Step title="Write Event Description">
    Use the rich text editor to describe your event:
    
    **What to Include:**
    - What the event is about
    - Who should attend
    - What to expect
    - What to bring (if anything)
    - Contact information for questions
    
    **Structure Template:**
    1. Brief overview of the event
    2. Detailed agenda or activities
    3. Target audience
    4. Practical information (parking, refreshments, etc.)
    5. Contact details for questions
    
    <img src="/images/content/event-description-editor.png" alt="Rich text editor for event description" />
  </Step>

  <Step title="Configure Registration">
    Set up how people can register:
    
    **Registration Required** (Toggle)
    - Turn on if people need to register
    - Turn off for drop-in events
    
    **Registration URL** (If registration required)
    - Link to registration form
    - External booking system
    - Email address for registration
    - Phone number for bookings
    
    <img src="/images/content/event-registration.png" alt="Registration settings with toggle and URL field" />
    
    **Registration Examples:**
    - "mailto:events@mmsppg.org"
    - "https://eventbrite.com/your-event"
    - "tel:01234567890"
    - "https://forms.google.com/your-form"
  </Step>

  <Step title="Add Featured Image">
    Choose an image to represent your event:
    
    1. Click the **Featured Image** field
    2. Upload a new image or select existing
    3. Add descriptive **Alt Text**
    4. Consider how it looks as a thumbnail
    
    <img src="/images/content/event-featured-image.png" alt="Featured image upload for events" />
    
    **Image Ideas:**
    - Photo from previous similar events
    - Generic health/community images
    - Event poster or flyer
    - Location photo
  </Step>

  <Step title="Set Category and Status">
    Organize and control visibility:
    
    **Category** (Optional)
    - Group similar events together
    - Makes browsing easier for visitors
    - Helps with organization
    
    **Status** (Required)
    - **Draft**: Still working on it
    - **In Review**: Ready for approval
    - **Published**: Live on the website
    
    <img src="/images/content/event-category-status.png" alt="Category and status selection fields" />
  </Step>

  <Step title="Save and Publish">
    Choose your next action:
    - **Save as Draft**: Continue working later
    - **Submit for Review**: Send to reviewers
    - **Publish**: Make it live (if you have permission)
    
    <img src="/images/content/event-save-publish.png" alt="Save and publish options for events" />
  </Step>
</Steps>

## Event Fields Explained

### Required Fields

<Tabs>
  <Tab title="Title">
    **Purpose**: The name of your event as it appears to visitors
    
    **Best Practices**:
    - Be specific and descriptive
    - Include the type of event
    - Add date/month if it's a recurring event
    - Keep it under 60 characters when possible
    
    **Good Examples**:
    ‚úÖ "Monthly PPG Meeting - March 2024"
    ‚úÖ "Diabetes Management Workshop"
    ‚úÖ "Community Health and Wellbeing Fair"
    
    **Avoid**:
    ‚ùå "Meeting"
    ‚ùå "Workshop"
    ‚ùå "Event This Month"
  </Tab>

  <Tab title="Event Date">
    **Purpose**: When the event takes place
    
    **Guidelines**:
    - Use the date picker for accuracy
    - Double-check the date is correct
    - Consider lead time for promotion
    - Think about conflicts with holidays
    
    **Planning Tips**:
    - Give at least 2 weeks notice for most events
    - Check local school holidays
    - Avoid major religious holidays
    - Consider seasonal factors
  </Tab>

  <Tab title="Description">
    **Purpose**: Detailed information about the event
    
    **What to Include**:
    - Event overview and purpose
    - Agenda or schedule
    - Target audience
    - What attendees will gain
    - Practical details (refreshments, parking, etc.)
    
    **Structure Example**:
    ```
    Join us for our monthly PPG meeting where we discuss 
    practice improvements and community health initiatives.
    
    Agenda:
    - Practice updates
    - Patient feedback review
    - New service planning
    - Community health projects
    
    This meeting is open to all registered patients. 
    Light refreshments will be provided.
    ```
  </Tab>

  <Tab title="Status">
    **Purpose**: Controls whether the event is visible to the public
    
    **Status Options**:
    - **Draft**: Work in progress, only you can see it
    - **In Review**: Submitted for approval
    - **Published**: Live on the website
    
    **Workflow**:
    Draft ‚Üí In Review ‚Üí Published
    
    **When to Use Each**:
    - Use Draft while planning details
    - Use In Review when ready for feedback
    - Use Published when event is confirmed
  </Tab>
</Tabs>

## Managing Event Registration

### Registration Options

<AccordionGroup>
  <Accordion title="üìß Email Registration">
    **When to Use**: Simple events with basic registration needs
    
    **Setup**:
    - Use a dedicated email address
    - Set up auto-reply confirmation
    - Create a simple tracking system
    
    **Example**: "Register by emailing events@mmsppg.org with your name and contact details"
    
    **Pros**: Simple, personal touch
    **Cons**: Manual tracking required
  </Accordion>

  <Accordion title="üìû Phone Registration">
    **When to Use**: Older demographic, complex requirements
    
    **Setup**:
    - Designate specific phone hours
    - Train staff on registration process
    - Keep written records
    
    **Example**: "Call 01234 567890 between 9 AM - 5 PM to register"
    
    **Pros**: Personal service, immediate answers
    **Cons**: Limited availability, staff time required
  </Accordion>

  <Accordion title="üìù Online Forms">
    **When to Use**: Detailed information needed, automated processing
    
    **Setup**:
    - Use Google Forms, Microsoft Forms, or similar
    - Include all necessary fields
    - Set up automatic confirmations
    
    **Example**: Link to a form collecting name, contact details, dietary requirements, etc.
    
    **Pros**: Automated, detailed data collection
    **Cons**: Requires technical setup
  </Accordion>

  <Accordion title="üé´ External Booking Systems">
    **When to Use**: Paid events, complex ticketing needs
    
    **Options**:
    - Eventbrite
    - Ticketmaster
    - Local booking systems
    
    **Example**: "Book tickets at eventbrite.com/your-event"
    
    **Pros**: Professional ticketing, payment processing
    **Cons**: May have fees, less control
  </Accordion>
</AccordionGroup>

## Troubleshooting Common Issues

<AccordionGroup>
  <Accordion title="üö´ Low Registration Numbers">
    **Possible Causes:**
    - Insufficient promotion time
    - Unclear event benefits
    - Inconvenient timing
    - Complex registration process
    
    **Solutions:**
    - Start promotion earlier
    - Highlight clear benefits and outcomes
    - Survey audience for preferred times
    - Simplify registration process
    - Use multiple promotion channels
  </Accordion>

  <Accordion title="üìÖ Date/Time Conflicts">
    **Prevention:**
    - Check local event calendars
    - Avoid school holidays and major events
    - Survey regular attendees for preferences
    - Keep a master calendar of all events
    
    **If Conflicts Arise:**
    - Communicate changes clearly
    - Offer alternative dates if possible
    - Apologize for any inconvenience
    - Learn for future planning
  </Accordion>

  <Accordion title="üìç Venue Issues">
    **Common Problems:**
    - Venue unavailable
    - Capacity too small/large
    - Accessibility concerns
    - Technical equipment failures
    
    **Solutions:**
    - Always have backup venue options
    - Visit venues before booking
    - Confirm bookings in writing
    - Test equipment beforehand
    - Have contingency plans
  </Accordion>

  <Accordion title="üí¨ Communication Problems">
    **Issues:**
    - Attendees not receiving information
    - Last-minute changes not communicated
    - Registration confirmations not sent
    
    **Prevention:**
    - Use multiple communication channels
    - Keep contact lists updated
    - Set up automatic confirmations
    - Have a communication timeline
    - Designate communication responsibilities
  </Accordion>
</AccordionGroup>
